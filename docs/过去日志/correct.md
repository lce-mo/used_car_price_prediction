# 实验日志 2：本地/线上 MAE 差距诊断与口径修正

## 记录目的

本文件用于补充并修正 `experiments/experiment_log.md` 中关于本地 MAE 与线上 MAE 的解释。

`experiment_log.md` 里的大量历史实验仍可作为“旧口径下的相对排序参考”，但其本地 MAE 绝对值已经不能再作为真实本地分数使用。核心原因不是典型过拟合，而是原始数据读取方式错误导致部分样本列错位。

## 结论摘要

- 旧日志中的本地 `400+ / 450+` MAE 主要是错位数据上的验证分数，不是真实训练标签 `price` 上的 MAE。
- 线上 `700+` MAE 与“旧 OOF 预测对齐正确 `price` 后重算”的 MAE 非常接近。
- 因此“本地 400 多、线上 700 多”的主因不是模型过拟合，而是本地评估口径错误。
- 旧实验中“本地下降 5，线上也大概率下降”的现象说明旧口径仍有一定排序相关性，但不能再信任其绝对数值。
- 从本文件之后，所有新实验必须使用正确读取方式重跑，并重新建立本地/线上对应关系。

## 与 `experiment_log.md` 的关键差异

| 项目 | `experiment_log.md` 历史口径 | 本日志修正口径 |
| --- | --- | --- |

## 数据读取问题

赛题原始文件是空格分隔，但部分缺失字段表现为连续空格。使用 `sep=r"\s+"` 会把连续空格合并，导致缺失字段被吞掉，后续字段整体左移。

正确读取方式：

```python
pd.read_csv(path, sep=" ")
```

错误读取方式：

```python
pd.read_csv(path, sep=r"\s+")
```

## 旧 OOF 按真实 `price` 重算

为了判断线上/本地差距是否来自过拟合，我将旧 OOF 预测按 `SaleID` 对齐到正确读取的真实 `price`，重新计算 MAE。

| OOF 文件 | 旧日志/旧文件 MAE | 对齐正确 `price` 后 MAE |
| --- | ---: | ---: |
| `outputs/test/full150000_modelte_s50_repeated/oof_predictions.csv` | `462.5448` | `750.1727` |
| `outputs/full150000_log_sqrt_fine_blends/best_blend_oof_predictions.csv` | `456.4064` | `744.0000` |
| `outputs/full150000_multi_blend_search/best_blend_oof_predictions.csv` | `455.1386` | `742.7397` |

这个结果解释了为什么线上 MAE 一直在 `700+`：线上分数更接近真实标签口径，而不是旧日志里的错位标签口径。

## 对旧线上记录的重新解释

`experiment_log.md` 中的线上记录仍然有参考价值，因为线上是平台按真实测试标签评估。

但旧本地分数需要重新解释：

| 历史现象 | 旧解释 | 现在解释 |
| --- | --- | --- |
| 本地 `486`，线上 `744` | 可能存在严重过拟合或 test_B 分布更难 | 主要是本地标签错位，旧本地 MAE 被低估 |
| 本地融合从 `462` 到 `455`，线上也变好 | 融合有效 | 融合可能确实有效，但旧本地绝对值无效 |
| 线上约 `730` | 与本地差距约 `270` | 与旧 OOF 对真实 `price` 重算后的 `742.7` 接近 |

因此，历史结论中“哪个方向可能有效”可以暂存，但必须用修复后的读取方式重新验证。

## 后续实验规范

从本日志之后，所有新实验必须遵守：

1. 原始赛题文件统一使用 `sep=" "` 读取。
2. 不再引用旧日志里的本地 MAE 绝对值作为正式判断。
3. 所有候选模型必须重新生成 OOF，并用正确 `price` 计算 MAE。
4. 新提交文件必须基于修复读取后的重新训练结果生成，不能复用旧 submission 作为“修复版”。
5. 旧实验只保留以下用途：
   - 看大方向是否可能有用
   - 看线上历史提交趋势
   - 作为排查记录
6. 新实验日志应明确标注“correct parser”或“修复读取后口径”。

## 下一步建议

优先级从高到低：

1. 用修复后的 `src/train.py` 重跑当前最强单模型配置，建立新的本地 CV 基线。
2. 重新生成 `test_B` submission，并提交一次验证新口径线上表现。
3. 重跑 `log1p`、`sqrt`、`log_s10`、`log_s20` 成员，重新搜索融合权重。
4. 只有在新口径下确认有效后，再恢复目标编码、切片分析、校准等后续优化。

当前最重要的不是继续调参，而是先重建可信的本地评估体系。
