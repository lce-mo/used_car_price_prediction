# 实验日志

## 说明

- 评估指标：`MAE`
- 当前最佳的小规模方案：`E3`
- 小规模实验统一使用：训练集 `20000` 行、测试集 `5000` 行、`2-fold`
- 所有标记为“`小实验暂不优`”或“`待复核`”的结论，都需要在更大样本和更多折数下再次验证

- 当前绝对误差贡献最高的交叉异常子群体是 `power=0 & date_invalid_any`，但只占总绝对误差约 `0.42%`
- 异常群体整体并不是当前主版本的真实瓶颈，说明“异常清洗”不应升级为主线
- 真正值得警惕的是少量“高价且异常”的离群个体，例如 `power=0` 但价格在 `3万~8万` 的样本
- 因此异常方向当前应降级为次主线：若后续继续沿此方向尝试，只建议做轻量异常标记/修正特征，不建议对整类异常样本统一清洗或加权
- 主线仍更可能回到围绕 `model` 信息做更稳的层级编码或低频回退方案

## 第十三轮 `model` 编码收益拆解分析

统一说明：

- 本轮不新增训练，只复用已有全量 OOF 结果做拆解
- 对比对象固定为：
  - 上一个正式版本：`W5`
  - 当前正式版本：`W5 + model OOF 均价编码`
- 目的是判断 `model` 编码到底主要帮助了哪些 `model` 及哪些频次区间

核心观察：

- `model` 编码收益最明显的并不是极低频 `model`，而是中高价、样本量中等及以上的 `model`
- 收益最大的若干 `model` 包括：
  - `218`：MAE 改善约 `1138.80`
  - `167`：改善约 `663.94`
  - `206`：改善约 `559.42`
  - `222`：改善约 `466.17`
  - `189`：改善约 `465.54`
- 这些收益最高的 `model` 基本都不是极低频，且价格层级明显高于总体平均
- 按 `model` 频次分桶看：
  - `<10`：当前反而变差约 `76.90`
  - `10-19`：改善约 `60.24`
  - `20-49`：改善约 `121.69`
  - `50-99`：改善约 `136.87`
  - `100+`：改善约 `63.85`
- 在高价样本（价格前 `20%`）里，这个规律更明显：`20-49`、`50-99`、`100+` 频次段都出现了稳定正收益，而 `<10` 仍为负收益

方向结论：

- 当前 `model` 编码的核心价值，不是“拯救极低频尾部”，而是帮助模型更好地区分中高价、结构相对稳定的车型层级
- 这也解释了为什么：
  - 粗暴低频回退会变差
  - 低频标记会变差
  - 因为当前收益主场不在“极低频 `model`”
- 因此后续如果继续沿 `model` 方向优化，更值得考虑：
  - 围绕中高价 `model` 的更稳定表达
  - 或只对极少数极低频 `model` 做非常克制的处理
  - 而不是把“低频 `model` 问题”当成主线

## 第十四轮当前主版本价格×车龄主损失来源分析

统一说明：

- 本轮不新增训练，只复用已有全量 OOF 结果做分析
- 主对象是当前线上主版本 `W5 + model OOF 均价编码`
- 上一个正式版本 `W5` 仅作为改善对照
- 主切片固定为 `价格分桶 × 车龄分桶`

核心观察：

- 当前总绝对误差贡献最高的前三个格子是：
  - `Q5 × 8y_plus`：当前 MAE `1434.31`，贡献占比约 `16.23%`，相对 `W5` 改善约 `178.35`
  - `Q4 × 8y_plus`：当前 MAE `595.32`，贡献占比约 `16.19%`，相对 `W5` 改善约 `77.80`
  - `Q5 × 5_8y`：当前 MAE `1095.70`，贡献占比约 `14.76%`，相对 `W5` 改善约 `171.59`
- 这说明当前主战场已经不再是“高价新车”，而是“高价老车 / 高价中老龄车”
- 一维上看：
  - 价格分桶贡献最高的是 `Q5`
  - 车龄分桶贡献最高的是 `8y_plus`
- 虽然 `Q5 × 0_1y` 的单格 MAE 仍然很高，但由于样本量占比很小，对总 MAE 的贡献已明显低于高价老车格子

方向结论：

- 后续特征工程不应再笼统围绕“高价新车”展开
- 更值得优先围绕：
  - 高价老车
  - 高价中龄车
  - 高价区间内的车型层级差异
- 这也和前一轮结论一致：当前 `model` 编码真正带来收益的主场，更像是中高价、结构更稳定的车型，而不是极低频或准新车尾部

## 误差分析摘要

基于当前最佳小规模 baseline，可以观察到：

- 高价车的绝对误差明显更大
- 车龄越新的样本，误差通常越大
- 部分均价较高的品牌更难预测

这说明后续特征工程应优先围绕“提高高价值车辆的区分能力”展开。
## 第十五轮：`model × age_bin` 非目标统计特征 A/B

实验目的：
- 不再扩展目标编码变体，转而验证一组低风险的 `model × age_bin` 非目标统计特征，重点服务当前主损失来源“高价老车 / 高价中龄车”。
- 只做单一 A/B 对照，避免和其他新特征、参数改动混在一起。

实验设计：
- 训练口径：`50000 / 3-fold / sample_seed=2024 / predict-test=false`
- 基线：当前中规模强基线 `M2 + LightGBM + model OOF target encoding(smoothing=20)`
- 参数：`learning_rate=0.07, n_estimators=2150, num_leaves=287`
- 新增特征组（独立开关）：
  - `model_age_count`
  - `model_age_count_ratio`
  - `model_age_power_median`
  - `model_age_kilometer_median`
- `age_bin` 沿用现有口径，`__AGE_MISSING__` 单独成桶
- 当 `model × age_bin` 样本数 `< 20` 时，仅对连续统计值回退到 `model` 级别中位数

结果记录：
- 基线：`outputs/test/model_te_seed2024_recheck_50k`
  - `3-fold MAE = 545.09`
  - `mae_std = 5.63`
- 实验组：`outputs/test/model_age_group_stats_seed2024_50k`
  - `3-fold MAE = 546.53`
  - `mae_std = 6.23`
- 对比结论：
  - 新增 `model × age_bin` 非目标统计后，平均 MAE 变差约 `1.44`
  - 波动也略有放大，说明这组特征没有带来更稳的高价老车 / 中龄车刻画

当前判定：
- 这组 `model × age_bin` 非目标统计特征当前不保留
- 不进入全量复核，也不生成新的 `testB` 提交文件
- 学习结论：
  - 当前有效的 `model` 信息更像是“车型整体价格层级信号”
  - 再按 `age_bin` 细化后，新增统计没有补出有效信息，反而增加了噪声

## 第十六轮：高价老车/中龄车分段建模首版

实验目的：
- 不再继续加全局特征，改为验证“硬分段训练”能否直接压当前主损失来源。
- 首版只切最痛点子段：`Q5 × {5_8y, 8y_plus}`。

实验设计：
- 训练口径：`50000 / 3-fold / sample_seed=2024 / predict-test=false`
- 基线：当前中规模强基线 `M2 + LightGBM + model OOF target encoding(smoothing=20)`
- 参数：`learning_rate=0.07, n_estimators=2150, num_leaves=287`
- 分段方案：
  - 训练子段标签：按每折训练部分的 `price` 80% 分位阈值 + `age_bin in {5_8y, 8y_plus}`
  - 路由：先用全局模型预测价格，再结合 `age_bin` 判断是否进入子模型
  - 合成：命中路由则直接替换为子模型预测

结果记录：
- 基线：`outputs/test/seg_baseline_model_te_seed2024_50k`
  - `3-fold MAE = 545.09`
  - `mae_std = 5.63`
- 分段实验：`outputs/test/seg_q5_5plus_model_te_seed2024_50k`
  - `3-fold MAE = 564.34`
  - `mae_std = 6.01`
  - 相比基线变差约 `19.25`

路由与子段表现：
- 路由覆盖约 `11.6% ~ 12.2%`
- 路由 precision 约 `0.92 ~ 0.93`
- 路由 recall 约 `0.90 ~ 0.93`
- 说明首版失败不是因为路由完全错误，而是子模型本身没有优于全局模型
- 子段 MAE（分段后）约：
  - Fold1: `1590.91`，高于全局子段 MAE `1510.36`
  - Fold2: `1487.96`，高于全局子段 MAE `1417.76`
  - Fold3: `1544.05`，高于全局子段 MAE `1449.15`

当前判定：
- 首版“硬分段训练 + 直接替换”当前不保留
- 不进入全量复核，也不生成新的 `testB` 提交文件
- 学习结论：
  - 当前全局模型对高价老车/中龄车的刻画虽然误差仍高，但仍优于这版简单子段模型
  - 问题不在“是否切段”，而更可能在“子段内信息不够”或“直接替换过于激进”

## 2026-04-27：target transform 融合实验与线上结果

实验目的：
- 针对全量 OOF 误差切片中持续出现的高价车系统性低估，验证 `log1p(price)` 是否过度压缩高价差异。
- 不再继续 hard-routed expert、精准切片软权重、简单分桶校准；改为测试不同 target transform 的模型多样性，并用 OOF blending 判断是否互补。

固定配置：
- 模型：`LightGBM`
- 参数：`learning_rate=0.07`, `n_estimators=2150`, `num_leaves=287`
- 特征：`use_group_stats=true`, `use_model_target_encoding=true`, `target_encoding_smoothing=50`
- CV：`repeated_stratified`, `3 folds x 3 repeats`, `stratify_price_bins=5`
- 第一阶段数据：`data/raw/used_car_train_first50000.csv`
- 全量复验数据：`data/raw/used_car_train_20200313.csv`

first50000 结果：

| run | fold MAE | 聚合 OOF MAE | 目标五组 MAE | Q5 MAE |
| --- | ---: | ---: | ---: | ---: |
| `log1p` baseline | `550.10` | `521.39` | `773.16` | `1368.26` |
| `sqrt(price)` | `556.64` | `527.25` | `775.57` | `1365.32` |
| `price^0.75` | `562.87` | `532.35` | `778.86` | `1374.31` |
| raw `price` | `580.26` | `546.65` | `782.66` | `1386.02` |
| `0.6 * log1p + 0.4 * sqrt` | - | `515.39` | `762.42` | `1344.92` |

全量复验结果：

| run | fold MAE | 聚合 OOF MAE | 目标五组 MAE | Q5 MAE |
| --- | ---: | ---: | ---: | ---: |
| `log1p` baseline | `486.05 ± 7.56` | `462.54` | `685.01` | `1184.42` |
| `sqrt(price)` | `488.31 ± 6.37` | `465.23` | `683.01` | `1175.94` |
| `0.5 * log1p + 0.5 * sqrt` | - | `456.48` | `673.14` | `1160.16` |

关键验证：
- `sqrt(price)` 单模型整体不如 `log1p(price)`，但在 Q5 和目标高误差切片上有不同偏差，说明它不是替代模型，而是有效融合成员。
- first50000 上 OOF blend 最优为 `0.6 * log1p + 0.4 * sqrt`，聚合 OOF MAE 从 `521.39` 降到 `515.39`。
- 全量上 OOF blend 最优为 `0.5 * log1p + 0.5 * sqrt`，聚合 OOF MAE 从 `462.54` 降到 `456.48`。
- 全量 5 折 alpha 验证稳定，`blend_cv_mae=456.50`，不是单纯在同一份 OOF 上过拟合 alpha。

提交文件：
- `outputs/final_blend_log05_sqrt05_full_predict/submission.csv`
- 融合方式：`0.5 * final_log_s50_full_predict + 0.5 * final_sqrt_full_predict`

线上结果：
- 线上 MAE：`730.4803`
- 相比上一版约提升 `10 MAE`

结论：
- target transform 单模型替换失败，但 target transform 融合有效。
- 当前最有价值的方向不是继续加权或 hard expert，而是利用不同目标变换形成模型多样性。
- 下一步优先围绕融合扩展：搜索 `log/sqrt` 附近权重、加入不同 smoothing 的 `log1p` 成员，或再加入 CatBoost 等异构模型。
